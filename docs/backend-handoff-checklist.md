# 向真实后端团队索要的资料

对接真实后端时，优先要这些，不需要先要后端源码。

## 必要信息

前端交易保障区现在会展示“真实后端接入就绪”面板。下面这些资料会直接影响面板里哪些项目能从“演示可跑”变成“真实可用”。

- API 基础地址，例如 `https://api.xxx.com`
- API 文档，至少覆盖 [api-contract.md](api-contract.md) 中列出的接口
- 测试环境地址和生产环境地址
- 测试账号或测试钱包
- 登录方式：账号密码、短信、钱包签名、JWT、Session 或其他
- 鉴权头格式，例如 `Authorization: Bearer <token>`
- 钱包签名消息格式、验签方式和地址权限规则，例如是否使用 EIP-191 `personal_sign`、EIP-712 typed data 或服务端 nonce 登录
- CORS 跨域白名单，是否允许前端部署域名访问
- 文件/交付物上传方式，如果交付物不是纯文本
- 任务状态枚举和状态转换规则
- Agent 所有权证明方式，例如 Agent NFT、DID 签名、模型承诺哈希或代码仓库签名
- Agent 所有权校验状态字段，例如已校验、待人工复核、已拒绝、未提交，以及校验方式、审核时间和审核签名
- 创建托管任务时，后端如何阻止未校验或被拒绝的 Agent 进入执行方案
- 创建托管任务时，后端如何校验手选执行方案的剩余可承接额度、总质押覆盖和实时容量占用，并返回可读拒绝原因
- Agent 列表是否返回 `lockedCapacity`、`availableEscrowLimit` 或等价字段，哪些任务状态会占用容量
- Agent 服务边界字段如何审核和展示
- 创作者收益流水、提现状态、出账审核、钱包地址、争议冻结、质押锁定/解锁和真实出账状态如何返回
- 托管资金流水、支付流水号、链上交易哈希、合约事件或出账单号如何返回
- 争议理由、争议证据、仲裁员身份、裁决权限、裁决理由、裁决附件和申诉机制如何提供
- 错误码和错误响应格式

## 拿到地址后的第一步检查

拿到测试环境或生产环境 API 地址后，先在前端项目根目录运行：

```powershell
node tools/verify-api-contract.js https://api.xxx.com
```

如果后端接口需要登录 token：

```powershell
$env:QIANZHI_API_TOKEN="test-token"
node tools/verify-api-contract.js https://api.xxx.com
```

检查结果里：

- `失败` 表示当前前端不能可靠接入，需要后端修必需接口或核心字段。
- `警告` 表示基础展示可继续联调，但上线前仍要补齐推荐能力。
- `通过` 表示当前原型需要的基础合约已经满足。

## 建议直接发给对方的问题

1. 你们真实后端的 API Base URL 是什么？
2. 是否能兼容 `/api/health`、`/api/readiness`、`/api/agents`、`/api/tasks`、`/api/evidence`、`/api/creators`、`/api/withdrawals` 这组接口？
3. 登录和鉴权怎么做？前端需要带什么 header？
4. Agent 注册、任务发布、托管、验收、争议、结算分别对应哪些接口？
5. 返回字段是否能按 `docs/api-contract.md` 的结构提供，或者你们已有字段映射是什么？
6. 前端部署域名确定后，能否加入 CORS 白名单？
7. 是否有测试账号、测试钱包、测试任务数据？
8. 关键操作签名采用什么消息格式？后端如何防重放、验签、恢复地址，并判断该地址有权操作对应任务、提现单或 Agent？
9. Agent 所有权证明由谁校验？自动校验和人工复核分别走哪些接口，审核签名、审核备注和拒绝原因会返回什么字段？
10. Agent 入驻时是否会校验创作者声明：确认所有权、平台非托管、收益来自按结果任务结算？
11. 创建任务时，未校验或被拒绝的 Agent 会返回什么错误码和错误信息？
12. 手选 Agent 的剩余可承接额度、总质押覆盖或实时容量不足时，后端会返回什么错误码、缺口数值和可读提示？
13. 争议仲裁由谁裁决？争议理由、争议证据、裁决记录、附件、时限和申诉状态分别由哪些字段返回？
14. 创作者经营台需要的收益、提现、出账审核、争议冻结、质押锁定和 Agent 归属数据分别从哪些接口读取？
15. 提现申请、批准、拒绝和真实出账完成分别需要哪些签名、审核字段、钱包地址和支付流水编号？
16. 任务托管、协议费、创作者分账和退款分别对应哪些资金流水字段或链上事件？
17. 文件证据如何上传或登记？是否返回可复用的 `uri`、`hash`、权限策略、过期策略和下载授权字段？
18. 能否保证缺少争议证据、挑战窗口未结束自动结算、缺少仲裁理由或缺少证据核对说明时，后端拒绝状态流转并保持任务状态不变？

## 就绪面板对应的后端能力

- API 连接：`/api/health` 能返回环境、服务名和健康状态。
- 账号与鉴权：能提供登录、钱包签名或会话校验，并说明前端请求如何携带身份。
- 任务状态机：后端负责校验托管、接单、提交、验收、争议、仲裁和自动结算，不能只信前端按钮。
- 资金与出账：返回托管支付流水、协议费、创作者分账、退款、链上哈希或出账单号。
- 证据上传：提供交付物和争议证据上传接口，返回文件引用、哈希、权限和过期策略。
- 签名凭证：返回用户、Agent、仲裁员和平台系统操作签名或可审计凭证。
- 争议仲裁：返回仲裁员身份、裁决权限、裁决理由、附件、时限和申诉状态。
- 提现与结算：返回创作者可提现余额、提现申请、审核、拒绝、批准、真实出账和争议冻结状态。

如果真实后端能提供 `/api/readiness`，前端会优先使用后端返回的 `capabilities` 作为交易保障区的就绪状态；如果暂时没有这个接口，前端会继续用本地数据做演示推断。

## 前端拿到资料后要改哪里

主要改 `scripts/config.js`：

```js
window.QIANZHI_CONFIG = {
    apiBase: 'https://api.xxx.com',
    apiMode: 'required',
    environmentName: '真实业务后端'
};
```

如果真实接口路径或字段不一致，再改 `scripts/app.js` 里的 `apiRequest` 调用和数据归一化函数。
