# 当前版本交接说明

更新时间：2026-05-28

## 项目定位

当前前端已经从原来的“AI 商品商城展示页”调整为“AI Agent 任务交易平台原型”。

核心表达是：

- 我们不生产 Agent。
- 创作者拥有自己的 Agent，并把 Agent 注册进市场。
- 创作者的专业 Agent 解决通用 AI 难以稳定处理的垂直领域复杂问题。
- 用户发布任务，按结果付费。
- 普通用户不需要自己研究复杂 AI 工作流，只需要描述任务、验收标准和预算，由平台匹配 Agent 并降低使用门槛。
- 平台负责匹配、托管、状态记录、验收、争议和结算安全。

### 前端逻辑

- 顶部钱包按钮会优先连接浏览器钱包；如果浏览器没有钱包插件，会提示继续使用演示签名，而不是伪造已连接状态。
- 关键任务流转、提现申请/审核和 Agent 所有权复核会生成操作签名；钱包已连接时会尝试 `personal_sign`，并把 `walletAddress`、`walletSignature` 和签名消息随操作提交。
- 没有钱包时仍使用 `demo:{actor}` 演示签名，方便本地流程跑通，但页面会在交易保障区标明这不等于真实鉴权。
- 支持本地静态演示。
- 支持自动探测 API。
- 支持通过 `scripts/config.js` 切换真实业务后端。
- 支持基于 URL hash 的页面式视图切换，差异较大的功能区不会全部堆在同一个长滚动页面里。
- 支持通过 `#agent-Agent编号` 进入单个 Agent 资产档案页，保留当前资产上下文。
- 支持通过 `#task-任务编号` 进入单笔任务交易档案页，保留当前任务上下文。
- 支持在页面顶部展示当前运行模式和交易状态来源，让演示者知道状态机来自浏览器、本地 mock 还是真实后端。
- 支持按当前 Agent 和任务数据计算交易保障覆盖度，动态展示所有权校验、资金托管、证据覆盖、审计链、退款闭环和待处理风险。
- 支持在交易保障区展示真实后端接入缺口，帮助判断当前页面能否用于真实交易演示。
- 支持用户、Agent 创作者、仲裁员三种体验身份，并按身份启用任务操作。
- 支持平台系统体验身份，用于模拟挑战窗口结束后的自动结算。
- 支持首页角色入口和顶部身份切换共用同一套身份状态，进入创作者经营、任务托管或交易保障时不会出现页面视角和操作身份不一致。
- 支持任务状态操作：
  - 生成方案
  - 托管预算
  - Agent 执行
  - 提交交付物和证据包
  - 验收结算
  - 发起争议
  - 仲裁结算或退款
- 支持任务托管页按交易状态和当前身份待处理任务进行筛选，方便用户、创作者和仲裁员快速定位需要处理的交易。
- 支持任务托管页按当前身份聚合待处理行动队列，显示任务状态、预算、需要执行的动作和交易档案入口。
- 支持创作者注册 Agent。
- 支持创作者在注册 Agent 前预览准入状态，提前知道 Agent 是直接进入可信匹配池，还是需要平台人工复核。
- 支持创作者注册 Agent 前确认所有权、非托管和按结果结算声明；未确认时前端不会提交，mock API 也会拒绝入驻。
- 支持在 Agent 市场直接选择 Agent 加入执行方案，市场选择会同步到首页任务托管单。
- 支持 Agent 市场按当前筛选结果计算可信供给池概览，让用户在选择 Agent 前先看到所有权校验覆盖、质押规模和可承接容量。
- 支持 Agent 市场按所有权状态筛选供给，降低用户把未复核 Agent 误认为可直接托管的概率。
- 支持 Agent 市场按当前任务预算筛选可承接 Agent，预算变化会同步刷新市场安全概览。
- 支持 Agent 市场按未完成任务占用后的剩余容量判断可承接 Agent，避免已经被活跃交易锁住的 Agent 继续被过度匹配。
- 支持优先使用后端返回的 Agent 容量快照，真实后端接入时不会因为前端只加载了部分任务而误判容量。
- 支持 Agent 市场按当前任务类型、预算、所有权、剩余可承接额度和质押覆盖筛选适配 Agent，降低下单前误选风险。
- 支持在 Agent 卡片上直接解释当前任务适配结果，让用户知道不适配原因来自类型、所有权、剩余可承接额度还是质押覆盖。
- 支持展示 Agent 所有权证明和服务边界。
- 支持展示 Agent 所有权校验状态，DID、Agent NFT、模型承诺哈希和代码仓库签名会标记为已校验。
- 支持平台对待人工复核的 Agent 所有权证明执行通过或拒绝，并保留审核签名。
- 未通过所有权校验的 Agent 不能进入任务自动推荐、手动托管方案或 mock API 任务创建。
- 支持按 Agent 聚合资产档案，让用户和创作者看到单个 Agent 的收入、风险、容量和历史任务。
- 支持按创作者聚合经营状态，让创作者看到自己拥有的 Agent 如何产生收益和承担风险。
- 支持创作者发起提现请求，平台身份可批准或拒绝；API 可用时会调用后端持久化，审核中和已批准未出账金额会从可提现余额里锁定。
- 支持任务发布前预览托管预算拆分、推荐 Agent、剩余可承接额度、质押覆盖和所有权校验。
- 支持在任务发布前预览结果契约，明确达标放款、未达标争议和平台只做托管/证据/结算安全。
- 支持为任务生成演示型契约指纹，并把指纹写入确认凭证、接单承诺、验收确认和结算展示。
- 支持用户在托管预算前逐项确认交易规则，确认后才推进到 `FUNDED` 状态并写入审计链。
- 支持 Agent 创作者在接单执行前逐项确认执行责任，确认后才推进到 `IN_PROGRESS` 状态并写入审计链。
- 任务卡片会展示结果验收标准和挑战窗口。
- 平台自动结算入口会检查挑战窗口；窗口未结束时不会作为待处理动作展示，也不能在本地演示状态机中提前结算。
- 任务卡片会展示托管前风控检查，包含 Agent 可用承接额度、质押覆盖、所有权证明、验收契约和证据状态。
- Agent 提交交付物时会先进入证据提交台，至少填写一条证据引用，前端生成演示哈希后再提交给状态机。
- 证据提交台新增本地文件哈希存证入口：选择文件后由浏览器计算 SHA-256，mock API 保存文件名、大小、哈希和 `qz://evidence/...` 引用，但不上传或保存原文件内容。
- 用户验收结算时会先进入放款确认清单，至少确认契约、证据、不争议和分账释放后才结算。
- 用户发起争议时会先进入争议证据提交台，至少填写一条争议证据，仲裁中心会展示争议理由和证据引用。
- 任务卡片会展示 Agent 交付证据和用户争议证据。
- 仲裁中心会集中展示争议任务，并要求仲裁员填写裁决理由和证据核对说明后，才能完成支持 Agent 或支持用户的裁决。
- 任务结算后会展示结算凭证或退款凭证，包含触发原因、托管预算、协议费、创作者分账、退款金额、证据数量和关联流水编号。
- 任务资金动作会展示为资金流水，包含事件类型、金额、交易编号、时间和对手方。
- 任务结算后会同步刷新创作者收益台账。
- 任务卡片会展示完整交易审计链，包含每次状态变化的角色、时间、资金/证据/结算摘要和审计编号。
- 关键任务操作会生成操作签名凭证，本地审计链会展示签名编号。
- 任务详情弹窗支持直接执行当前角色允许的任务操作，并在操作完成后刷新交易档案。
- 任务详情弹窗进入交付证据或争议证据提交台后，可从顶部或底部返回交易档案；提交成功后会重新打开最新交易档案。
- 任务最终结果会展示 Agent 声誉、成功率、成交数和质押风险变化。

### Mock 服务

`server.py` 只作为前端开发和演示用，不是最终业务后端。

它目前提供：

- 静态页面托管
- `/api/health`
- `/api/readiness`
- Agent 查询与注册
- `/api/creators` 创作者经营台聚合数据
- `/api/withdrawals` 提现审核队列查询、申请、批准和拒绝
- Agent 所有权证明、所有权校验状态与服务边界持久化/展示
- Agent 入驻会校验创作者声明：确认所有权、平台非托管和按结果结算
- Agent 所有权复核状态、复核说明、复核时间和复核签名持久化/展示
- 任务创建与状态流转
- 预算托管可保存前端传入的托管确认摘要，并写入任务历史和资金流水说明
- Agent 接单执行可保存前端传入的接单承诺摘要，并写入任务历史
- 用户验收可保存前端传入的验收确认摘要，并写入任务历史和结算凭证
- 资金流水 `escrow_events` 持久化，用于展示托管、协议费、分账和退款事件
- 结果验收标准与挑战窗口持久化
- 创建任务时会校验匹配 Agent 的剩余可承接额度和总质押覆盖，风险过高会返回 `422`
- 创建任务时会扣除 Agent 在活跃任务中的已锁容量，剩余可承接额度不足会返回 `422`
- 创建任务时会校验所有匹配 Agent 的所有权状态，未校验或被拒绝会返回 `422`
- 任务操作角色校验，错误身份会返回 `403`
- 任务状态流转需要携带操作签名，缺失或不匹配会返回 `401`
- 任务状态流转如果携带钱包签名字段，mock API 会保存 `walletAddress`、`walletSignature` 和 `message` 作为审计凭证；它只做字段保留，不做真实验签。
- 每次任务状态变化都会写入 `task_history`，前端会渲染为交易审计链
- Agent 提交交付物时需要提供证据包元数据
- `/api/evidence` 可登记文件哈希存证元数据，返回可写入交付或争议证据包的引用
- 用户发起争议时必须保存争议理由和争议证据
- 用户发起争议时必须提供争议理由和至少一条证据引用
- 挑战窗口未结束时自动结算会被拒绝，窗口结束后平台可自动结算
- 结算或退款结果持久化为可展示凭证
- 结算或退款会回写 Agent 成交数、声誉、成功率和质押风险
- 创作者提现申请会校验可提现余额并写入 SQLite，平台审核会保存审核签名和状态
- 默认种子数据包含一笔已结算演示任务和一笔争议演示任务，用于展示收益台账和仲裁中心
- SQLite 本地持久化
- 基础状态机校验

## 和原始版本的主要区别

原始版本：

- 更像 AI 商品/资源商城。
- 包含智能体、数据资产、算力、API 等商品。
- 部分资源缺失，部分交互会因为 DOM 缺失报错。
- 没有明确任务交易流程。
- 没有真实后端对接配置。

当前版本：

- 聚焦 Agent 任务交易。
- 以任务托管和按结果付费为核心，明确不做月费订阅大平台。
- 前端可独立打开，也可连接 mock API。
- 后续可以通过配置接入真实业务后端。
- 文档已经把 API、状态机、数据模型和交接资料列清楚。
- 新增本地 API 合约检查脚本，拿到真实后端地址后可以先跑检查再改前端配置；检查器现在会验证缺失或不完整 `creatorDeclaration` 的 Agent 入驻请求必须被拒绝，也会验证文件哈希存证、缺少 `signature`、错身份签名、签名动作不匹配、签名任务不匹配、缺少争议证据、挑战窗口未结束自动结算和缺少仲裁依据的任务状态流转必须被拒绝且不能改变任务状态。
- 新增视觉 QA 脚本，能用浏览器自动检查桌面/手机、浅色/深色、主要页面和证据弹窗的横向溢出、元素越界和文字裁切。

## 下一步依赖

如果要接真实后端，需要对方提供：

- API Base URL
- 接口文档
- 测试账号或测试钱包
- 登录/鉴权方式
- CORS 白名单配置
- Agent、任务、托管、争议、结算接口
- 创作者提现、出账审核、争议冻结和钱包支付状态接口
- 字段映射说明
- 能通过 `tools/verify-api-contract.js` 的测试环境地址

如果继续打磨前端，可以优先做：

- 移动端细节检查
- 真实年轻用户视角的视觉微调
- 真实后端验签、登录态绑定和钱包地址权限校验
- 真实文件上传、加密存储、权限控制和不可篡改存证服务

如果要发给别人看，可以优先选：

- 纯静态演示：部署 `index.html`、`styles/`、`scripts/`、`images/`，并把 `apiMode` 设为 `off`。
- 静态网页 + 测试后端：部署前端后，把 `apiBase` 指向公网测试 API，并把 `apiMode` 设为 `required`。
